## 4. 开题报告确定的研究主线

研究目标：在联邦学习场景下引入差分隐私保护，并通过多种优化策略联合设计，缓解差分隐私噪声导致的模型性能下降，在给定隐私预算约束下尽可能提升模型效用。

核心问题：

- 联邦学习虽然不直接共享原始数据，但模型更新仍可能泄露隐私。
- 差分隐私可以提供可量化隐私保护，但噪声会导致精度下降和收敛变慢。
- 现有方法多从单一角度优化，如裁剪、噪声、压缩，缺少对多模块耦合关系的系统研究。

开题报告中确定的三个核心优化方向：

- 梯度压缩：选择性上传重要参数，减少需要扰动的参数量。
- 自适应裁剪：根据训练动态调整裁剪阈值，降低固定裁剪造成的偏差或过量噪声。
- 异构噪声分配：根据参数重要性分配不同噪声强度，尽量保护关键参数更新质量。

## 6. 当前需要注意的代码事实

当前主入口是 `train.py` + `configs/base.yaml`。此前的 `run_ablation.py` 和 `run_hyperparam.py` 仍使用旧版 dataclass 配置风格，与当前 `FLTrainer(cfg)` 的 OmegaConf 接口不一致，虽然语法检查通过，但不建议直接作为最终实验脚本使用。

论文和实验中建议使用如下命名：

- `FedAvg`：非隐私联邦学习上界。
- `DP-FedAvg`：固定裁剪 + 均匀高斯噪声的隐私基线。
- `DP-FedSAM`：引入 SAM 本地优化的隐私基线，可作为可选强基线。
- `Ours-Core`：自适应裁剪 + Top-k 梯度压缩 + 异构噪声分配，不开启 residual 和 contrastive/fedprox 增强。
- `Ours-Full`：在 Ours-Core 基础上开启 residual、contrastive 或 fedprox、重要性模板/冻结等完整机制。
